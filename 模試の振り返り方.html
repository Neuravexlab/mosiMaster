模試の本当の目的

模試を受け終わったあと、何をしていますか？答え合わせをして、間違えた問題の解説を読んで、ノートに写して、終わり。この振り返り方では、次の模試で同じミスを繰り返します。

模試の本当の目的は、「今の自分の実力と、ゴールとの差を把握して、明日からの勉強を変えること」です。点数そのものに意味があるのではなく、その点数を生み出した「自分の思考プロセスのどこに穴があるか」を特定することに意味があります。

模試は「テスト」ではなく「診断」である

結果に一喜一憂するのではなく、「今後の方針を決めるための材料」として使う

解けなかった問題は「失敗」ではなく「自分の穴が見えたラッキー」

この意識で模試に向き合えると、振り返りの質が根本から変わります。

よくある振り返りのNG

よくあるNGなぜダメか
解説を読んで「なるほど」と思って終わる「分かった」と「自分で解ける」は別物。同じ問題を自力で解き直すまでやっていない
間違えた問題の解答をノートに丸写しする写す作業は「解いた」に含まれない。手が動いていても頭が動いていない。きれいなノートは芸術点のみ
「計算ミスだった」「凡ミスだった」で終わるなぜその計算ミスをしたのかを言語化していない。原因を特定しないと次も同じミスを繰り返す
点数の推移をグラフ化して管理する問題の難易度が毎回違うので実力が上がっているか下がっているか判別できない。メンタルが乱れるだけ
全問題を丁寧に復習する時間の浪費。自信を持って解いた問題を何度も復習する必要はない。「怪しかった問題」に絞る
模試の翌週以降にじっくり復習する記憶が薄れるほど復習の時間がかかる。当日〜翌日に、試験中の思考が鮮明なうちに行う

点数別：振り返りの深さを変える

点数によって、やるべき振り返りの深さが変わります。全員が同じ振り返りをする必要はありません。

6割以下の場合：深い分析より基礎に戻る
マークシート式で6割以下の場合、正直に言えば「勘や運で当たっている問題」が多数含まれています。この状態で1問1問丁寧に分析しても、分析している時間の方が無駄になります。 やるべきこと：
① ざっくり「どの分野が弱いか」を確認する（数学なら確率か微積か、英語なら単語か文法か）
② 「今の参考書のどこが甘いか」に気づく
③ 明日から基礎参考書に戻る決断をする

やらなくていいこと：
× 超難問を長時間かけて理解しようとする（今の実力では早い）
× 全問の解説を読み込む（時間対効果が低い）

振り返りにかける時間の目安：当日〜翌日の2時間以内
6〜8割の場合：プロセスを丁寧に分析する
「なんとなく分かっているのに取れない」問題が存在する層です。この層こそ、思考プロセスの分析が最も効果的に機能します。解けた問題と解けなかった問題の「どこが違うか」が鮮明に見えてきます。 やるべきこと：
① 間違えた問題・自信がなかった問題に絞って深く分析する
② 「解けなかった思考のどのステップで詰まったか」を特定する（後述）
③ 戦術の振り返り：時間配分・解く順番は正しかったか
④ 今後の方針を紙に書き出す

振り返りにかける時間の目安：当日〜翌日に2〜4時間
8割以上の場合：戦術と精度を磨く
知識・技術の土台はある層です。残りの2割の失点パターンを徹底的に潰すことと、本番に向けた「戦術」の最適化が中心になります。 やるべきこと：
① 失点した問題の「ミスの種類」を分類する（後述の4分類で）
② 正解した問題でも「より良い解き方はなかったか」を確認する
③ 時間配分・問題を解く順番・見直しのタイミングを最適化する
④ マーキングミスなど本番で起きうる事故を潰す

振り返りにかける時間の目安：当日〜翌日に1〜2時間（鮮度が命）

核心：思考プロセスを解体する

「解けなかった」で終わらせないための最大のポイントは、問題を解く思考プロセスを分解して、どのステップで詰まったかを特定することです。これは数学でも英語でも現代文でも歴史でも、すべての科目に共通する普遍的な構造です。

どんな問題を解くときも、頭の中ではこの6つのステップが動いています。「解けなかった」はこのどこかの話です。

ステップやること詰まっていたときの症状各科目での具体例
① 問題を
把握する		 「何を問われているか」「何が与えられているか」を正確に読み取る 「読んだのに何をすればいいか分からない」「問いと違うことを答えた」 数：条件の見落とし
英：設問の指示を誤読
現：「傍線部の説明」を求められているのに「筆者の主張」を答えた
歴：「なぜか」と「いつか」を混同
② 素材を
整理する		 与えられた情報・本文・資料を、使いやすい形に整理する（図・メモ・構造把握） 「問いは分かったが、どこから手をつけるか分からない」「頭の中が散らかっている」 数：図を描かずに頭だけで考えようとする
英：本文を読みながら情報を整理していない
現：段落の役割・論理構造を把握せず読む
歴：時代・人物・因果関係を整理しない
③ 翻訳する 問題の言葉・場面を「自分が知っている知識・解法・パターン」に結びつける 「知識はあったのに使えなかった」「解説を見て『あ、これか！知ってた！』となる」 数：「整数解を求めよ」→積の形に翻訳できなかった
英：接続詞「however」を見て逆接の論理を翻訳できなかった
現：「筆者の主張」を問われているのに傍線部周辺しか探さなかった
歴：「背景」を問われているのに「内容」を答えた
④ 方針を
立てる		 「この問いに答えるには何をどの順番でやるか」の道筋を決める 「翻訳はできたが、何から始めるか迷った」「複数の解き方があってどれを選ぶか分からなかった」 数：解法は分かるがどの公式から始めるか判断できなかった
英：先に設問を読むべきか本文を全部読むべきか判断できなかった
現：根拠を「本文から引く」か「自分の言葉で説明する」か迷った
歴：論述の構成（原因→経過→結果）が組み立てられなかった
⑤ 実行する 方針通りに、正確に実行する（計算・記述・選択肢の絞り込み・解答用紙への転記） 「方針は正しかったのに実行段階でのミスで失点」「答えに至る過程で間違えた」 数：符号ミス・移項ミス・計算ミス
英：正しい根拠を見つけたが選択肢の絞り込みを間違えた
現：根拠は分かっていたが記述が字数オーバーまたは不十分だった
歴：知識はあったが語句が不正確または漢字が違った
⑥ 吟味する 答えを出した後に、問いに正確に答えているか・指示・条件を満たしているかを確認する 「できたと思い込んでチェックしなかった」「時間切れで確認できなかった」 数：場合分け漏れ・単位ミス・問われているものと違う値を答えた
英：「本文に書かれていないもの」を選ぶ問いで本文にある選択肢を選んだ
現：字数制限・「〜を踏まえて」などの条件を満たしていなかった
歴：「〜の観点から述べよ」という指定を無視した

HOW 1問ごとにプロセスを特定する

間違えた問題を解いたときの頭の動きを思い出す（フロー図）
以下の問いかけを順番にたどっていくと、自分がどのステップで詰まったかが特定できます。 【全科目共通の診断フロー】

Q1：問題文を読んで「何をすればいいか」分かったか？
→ No → ①「把握」か③「翻訳」の問題
・問題文の意味自体が分からなかった → ①把握
・何を問われているかは分かったが、どう対処するか分からなかった → ③翻訳
→ Yes → Q2へ

Q2：手が動いたか（書き始められたか、本文を探し始められたか）？
→ No → ②「素材整理」か④「方針」の問題
・情報が多くて頭が混乱した → ②素材整理
・何をすべきかは分かったが順番・手順が組み立てられなかった → ④方針
→ Yes → Q3へ

Q3：実行の途中でミスをしたか？
→ Yes → ⑤「実行」の問題（計算ミス・選択肢の絞り間違い・語句の誤り）
→ No → Q4へ

Q4：答えを出して合っていると思ったが、実際は違っていたか？
→ Yes → ⑥「吟味」の問題（条件の見落とし・指示の見落とし）

Q5：時間が足りなかったか？
→ Yes → D「戦術ミス」（時間配分・解く順番を次回変える）
「知識がなかった」と「知識はあったが使えなかった」を区別する
この2つは全く別の問題です。混同すると対策が変わります。 「知識がなかった」場合：
→ 解説を読んで初めて「この定理があるんだ」と気づいた
→ 今日初めて見た内容だった
対策：参考書に戻って該当の知識を覚え直す

「知識はあったが使えなかった」場合：
→ 解説を見て「あ、これか！知ってた！」となった
→ 参考書では解けるのに、初見だと気づかなかった
対策：「この問題文のここを見たらこの解法を使う」という翻訳を言語化する

この違いを明確にしないまま「復習した」としても、同じ問題形式で必ず詰まる

ミスを4種類に分類する

間違えた問題を「バツ」で終わらせず、ミスの種類を4つに分類することで、対策が明確になります。

4種類のミスと対策
A 知識の穴
そもそも知らなかった・覚えていなかった
例：単語の意味を知らなかった / 公式を忘れていた / 用語の定義が曖昧だった
対策：参考書・単語帳に戻る。翌日・3日後・7日後で反復する

B 翻訳ミス
知識はあったが、問題文から正しい解法を引き出せなかった
例：「整数解を求めよ」を見ても因数分解もどきを思いつかなかった
対策：「この問題文のキーワードを見たらこの解法を使う」という翻訳を書き出す

C 実行ミス
方針は正しかったが、計算・記述の途中でミスをした
例：符号ミス / 移項ミス / 条件の書き漏れ / 場合分け忘れ
対策：自分のミスのパターンを記録する。計算ルーティンを増やす。解答の書き方を変える

D 戦術ミス
知識も方針も正しかったが、時間・順番・精神面で失敗した
例：この問題に時間をかけすぎた / 焦って読み飛ばした / マークをずらした
対策：次の模試での行動ルールを決める（「大問2が難しかったら3に飛ぶ」など）

振り返りの具体的なSTEP

HOW 当日〜翌日にやること

STEP 1｜試験中に「怪しかった問題」にマークをつけておく
試験本番中に、以下のマークを問題用紙につけておいてください。後で振り返るときに「どの問題に時間をかけるか」を判断できます。 ◎：自信を持って解いた（→振り返りは軽くでOK）
△：なんとなく解けたが自信がない（→優先的に振り返る）
✕：全く分からなかった・解けなかった（→プロセスを丁寧に分析）
？：時間がなくて解けなかった（→戦術の振り返り対象）

マークをつける習慣がないと、後で「あの問題はどう解いたか」を思い出せない。
試験中の思考が残っているうちに振り返ることが、復習時間を最大半分に短縮する。
STEP 2｜まず自己採点をして「△・✕・？」に絞る
試験後、まず自己採点をします。◎でつけた問題は答えが合っているかだけ確認して終わりです。振り返りの時間は△・✕・？の問題に集中させます。 注意点：
マーク式の場合、△でつけた問題が「合っていた」としても必ず振り返る。
なぜなら「たまたま正解した問題」は次回は解けないから。

振り返りの優先順位：
① ✕ で答えが間違っていた問題
② △ で答えが合っていたが自信がなかった問題
③ ？ で時間がなくて解けなかった問題（戦術の反省）
STEP 3｜1問ずつ「思考プロセスのどこで詰まったか」を特定する
解説を読む前に、まず自分でもう一度解いてみてください。解説を先に読んでしまうと「なぜ自分がそこで詰まったか」が分からなくなります。 手順：
① 問題をもう一度読む
② 解説を見ずに解いてみる（5分程度）
③ 詰まった場所を確認する：①〜⑥のどのステップか？
④ そこで詰まった理由を一言で書く
「公式を知らなかった（A）」
「この言葉から解法が思い浮かばなかった（B）」
「符号の処理を間違えた（C）」
⑤ 解説を読んで「正しい思考の流れ」を確認する
⑥ もう一度、解説を閉じて自力で解き直す
STEP 4｜「気づきの翻訳」を一言で書く
解き直しができたら、次の一文をノートに書いてください。これが「同じ問題形式で次回は必ず気づける」状態を作ります。 書く形式：
「問題文の（　　　）という表現を
（　　　）と読み替えれば、この解法に気づけた」

例：
「"整数解を求めよ"を見たら
"積の形に変形して約数を列挙する"と翻訳すれば良かった」

「"少なくとも1つ"という言葉を見たら
"余事象で考える"に即座につなげれば良かった」

この一文を書いた問題は、次回同じ形式が出たときに必ず気づけるようになる。
STEP 5｜戦術の振り返りを3行でまとめる
時間配分・解く順番・精神面について、「次の模試でこうする」を具体的に決めます。 書く形式（3行）：
今回失敗した戦術：「大問3に時間をかけすぎて大問5が解けなかった」
原因：「難しそうな問題で粘りすぎた」
次回のルール：「1問に5分かけても解けなかったら飛ばして最後に戻る」

模試は実戦練習の場。戦術を改善しながら本番に向けて精度を上げていく。

振り返りノートの作り方

振り返りノートの目的は「後でいつでも見返せること」と「同じミスを繰り返さないこと」の2つだけです。きれいなノートを作ることが目的ではありません。

HOW シンプルで使えるノートの形式

書くべき3つの要素だけ書く
【要素1】問題そのもの
問題をコピーして貼るか、問題番号と要点を書く
「解答を写す」は不要。問題文だけでOK

【要素2】詰まったプロセスと理由
「①〜⑥のどこで詰まったか」を一言で
例：「③翻訳：不定方程式と気づかなかった」

【要素3】気づきの翻訳
「次回この問題形式が出たら、こう考える」を一文で
例：「"整数の組"を見たら積の形に変形することを最初に試みる」

NG：解説をそのまま書き写す
NG：きれいに色分けして時間をかけすぎる
OK：手書きで雑でも「見返したときに分かる」程度に書く
科目別にインデックスをつける
次の模試の前に「英語のミスだけ見返す」ができるように、科目・日付だけはきちんと書いておきます。 ノートの端に：科目（例：数学ⅡBC）+ 日付（例：6/15前進マーク）

こうすることで：
次の模試の前日に「数学のミス一覧」をさっと見返せる
→ 「この前こういうミスをした」という意識で本番に臨める
ミスのパターンをリスト化する（自分のミス辞典を作る）
模試を重ねるうちに「自分が繰り返すミスのパターン」が見えてきます。それをリストにしておくと、模試の前の確認が効率化されます。 例）自分のミスパターンリスト：

【B：翻訳ミス】
・「整数解を求めよ」→ 積の形を作る（不定方程式の基本形）
・「少なくとも1つ」→ 余事象で考える
・「実数解の個数」→ 判別式またはグラフの位置

【C：実行ミス】
・負の数を二乗するとき符号を間違える → 必ず（）でくくる
・積分の計算で定数項の符号を落とす → 最後に展開して確認

このリストを模試の前日に3分で見返すだけで、防げるミスが大幅に減る。
📋 このやり方で3回模試を振り返った後のあなた

「自分のミスパターンリスト」が積み上がっています。模試の前日にそのリストを見返すと「あ、これ前回も同じミスをした」という項目がいくつかあることに気づく。そのミスを意識して本番に臨むと、防げるミスが明らかに減っている。模試が「受けて終わり」ではなく「次を変えるための材料」になっている状態です。

最後に：今後の方針を言語化する

振り返りの最後にやるべきことが1つあります。「今後の勉強をどう変えるか」を紙に書き出すことです。これをやらないと、振り返りをどれだけ丁寧にやっても「ただ感想を述べただけ」で終わります。

📝 振り返り後の方針シート（模試ごとに書く）
Q1今回の模試で一番よくできた科目・分野：
Q2今回の模試で一番よくできなかった科目・分野：
Q3A（知識の穴）のミスが多かった内容：
Q4B（翻訳ミス）のミスが多かった問題形式：
Q5C（実行ミス）で繰り返しているパターン：
Q6次の模試までに変える戦術（時間配分・解く順番）：
Q7明日から変えること（参考書・勉強法・時間配分）：
Q8次の模試での目標（点数より「何を改善するか」）：

「もっと勉強しよう」という感想で終わってはいけません。「どの科目の何を、どのやり方で、いつから変えるか」まで書いて初めて、模試の振り返りは完了です。

📌 今日のアクションプラン

今日受けた模試（または最近受けた模試）で、1問だけ振り返ってください。

やり方はこれだけです：

  1. 間違えた問題（または自信がなかった問題）を1問選ぶ
  2. 解説を見る前に、もう一度自力で解いてみる（5分）
  3. 「①〜⑥のどのステップで詰まったか」を一言で書く
  4. 「A知識・B翻訳・C実行・D戦術」のどのミスかを分類する
  5. 「次回この形式が出たら○○と考える」を一文で書く
  6. 解説を読んで、自力で解き直す

1問でいいです。まずこれだけ。これを毎回の模試で続けると、3回目には「自分のミスパターンリスト」ができています。そのリストが、本番で同じミスを繰り返さないための最強の武器になります。
